# Review Classification Policy

PR レビューの指摘を分類し、対応の優先順位を決定するためのポリシーです。

## 重要度の定義

- **Critical**: セキュリティ脆弱性、データ消失、致命的なバグ。即座に対応が必要。
- **P1 (High)**: 機能不全、仕様違反。マージ前に必ず修正。
- **P2 (Medium)**: パフォーマンス改善、コードの可読性。状況に応じて対応。
- **Nitpick (Low)**: タイポ、スタイルの好み。余裕があれば対応。

## 対応判断基準

1. **修正する**:
   - 明らかなバグやエラー。
   - セキュリティ上の問題。
   - プロジェクトの規約違反。
2. **保留/返信する**:
   - 誤検知、または現在のコンテキストでは不要な指摘。
   - 大規模なリファクタリングを伴い、本 PR の範囲を超えるもの。
   - スタイルの好みで、既存のコードベースと一貫しているもの。
